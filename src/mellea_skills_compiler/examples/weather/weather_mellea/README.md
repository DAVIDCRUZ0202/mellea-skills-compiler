# weather-mellea

Get current weather conditions and forecasts via [wttr.in](https://wttr.in/:help).

Generated by Melleafy 4.3.2 from `skills/weather/spec.md` (agent_skills_std runtime).

## Overview

- **Archetype**: D1 Integration — thin wrapper around the wttr.in public HTTP API
- **Modality**: synchronous_oneshot
- **Tool involvement**: P2 (pipeline calls wttr.in deterministically after intent classification)
- **Backend**: ollama / granite3.3:8b

## Prerequisites

- Python ≥ 3.11
- `curl` binary on PATH
- HTTP access to `wttr.in` (public service, no API key required)

## Installation

```bash
pip install -e .
```

## Usage

### CLI

```bash
weather-mellea "What's the weather in Dublin?"
weather-mellea "Will it rain in Tokyo tomorrow?"
weather-mellea "Week forecast for New York"
weather-mellea "Weather at ORD"
```

### Python

```python
from weather_mellea.pipeline import run_pipeline

result = run_pipeline(query="What's the weather in Paris?")
print(result)
```

## Supported Query Types

| Query type | Example | wttr.in endpoint |
|---|---|---|
| `current_summary` | "What's the weather in X?" | `?format=%l:+%c+%t+(feels+like+%f),+%w+wind,+%h+humidity` |
| `current_detailed` | "Detailed conditions in X" | `?0` |
| `forecast_3day` | "3-day forecast for X" | (default view) |
| `forecast_week` | "Week forecast for X" | `?format=v2` |
| `forecast_day0` | "Today's weather in X" | `?0` |
| `forecast_day1` | "Tomorrow's weather in X" | `?1` |
| `forecast_day2` | "Weather in X day after tomorrow" | `?2` |
| `json_output` | "JSON weather data for X" | `?format=j1` |
| `rain_check` | "Will it rain in X?" | `?format=%l:+%c+%p` |

## Out-of-scope queries

The skill rejects and redirects:
- Historical weather data
- Climate analysis or trends
- Hyper-local microclimate data
- Severe weather alerts
- Aviation/marine weather (METAR, etc.)
